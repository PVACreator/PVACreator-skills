# Gmail Account Object (GmailInfo)

Represents a Gmail account for listing and creation APIs.

## Field Reference

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `Id` | string | No (read-only) | Unique account ID generated by the system. |
| `CampaignName` | string | Yes (on add) | Name of the campaign that owns the account. |
| `ProxyType` | string | No | `HTTP` or `SOCKS5`. Defaults to `HTTP`. |
| `Proxy` | string | No | Proxy string `host:port:username:password`. |
| `CreateTime` | datetime | No (read-only) | Timestamp when the account entry was created. |
| `Message` | string | No (read-only) | Free-form status or failure reason. |
| `AccountStatus` | string | No (read-only) | `NotRegister`, `Registing`, `Success`, or `Fail`. |
| `CachePath` | string | No (read-only) | Local cache file path. |
| `ProfileInfo` | string | No (read-only) | Proxy/fingerprint summary such as `proxy: 127.0.0.1:8080, browserFinger: bind`. |
| `Username` | string | Yes | Gmail username. |
| `Password` | string | No | Account password. |
| `FirstName` | string | No | Given name. |
| `LastName` | string | No | Family name. |
| `BirthMonth` | string | No | Birth month `1`–`12`. |
| `BirthDay` | string | No | Birth day `1`–`31` (must be valid for the month). |
| `BirthYear` | string | No | Birth year, e.g., `1990`. |
| `Gender` | string | No | `1` = male, `2` = female. |
| `RecoveryEmail` | string | No | Recovery email. |
| `PhoneService` | string | Yes | SMS provider code: `0` (manual, do not use), `1` (smspva), `2` (5sim), `3` (smsactivate), `7` (Tinderre), `10` (Textverified), `11` (daisysms). Default `1`. |
| `PhoneApiKey` | string | Yes | API key for the SMS provider (required when `PhoneService != 0`). |
| `PhoneCountry` | string | No | Country code for the phone number. Values depend on the provider—see tables below. |
| `Enable_IMAP_POP3` | string | No | Non-empty value enables IMAP/POP3. Empty means disabled. |
| `PhoneNumber` | string | No (read-only) | Phone number obtained during registration. |
| `_2StepVerification` | string | No | `1` to enable two-step verification; anything else (or empty) disables it. |

## Notes When Adding Accounts

- **Required fields:** `Username`, `PhoneService`, and `PhoneApiKey` must be supplied (providers may additionally require `PhoneCountry`).
- **Optional fields:** The backend fills defaults or leaves them blank if omitted.
- **Read-only fields:** Do not include `Id`, `CreateTime`, `Message`, `AccountStatus`, `CachePath`, `ProfileInfo`, or `PhoneNumber` in the payload.
- **Proxy format:** `Proxy` must follow `host:port:username:password`, e.g., `proxy.example.com:8080:user:pass`.
- **PhoneCountry mapping:** Country codes depend on the SMS provider—see the following tables.
